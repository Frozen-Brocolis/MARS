# MARS

MARS - это проект по оркестрации multi-agent LLM-системы на базе `LM Studio`.
Основная идея проекта - улучшать качество ответа не одним большим промптом, а управляемым конвейером из специализированных агентов с проверкой, повторными итерациями, памятью и возможностью подключать поиск в интернете.

## Основная идея

Система рассматривает запрос пользователя как задачу, которая проходит через несколько ролей:

1. `Shield` защищает систему от prompt injection и попыток изменить поведение агентов.
2. `Planner` строит глобальный план решения.
3. `Manager` разбивает этапы плана на конкретные задачи.
4. `Worker` выполняет узкую, чётко поставленную задачу.
5. `Critic` оценивает результат и при необходимости отправляет задачу на повтор.
6. `Editor` собирает итоговый ответ из уже утверждённых материалов.

Такая архитектура рассчитана на рекурсивное или итеративное решение задач, где слабые промежуточные результаты можно дорабатывать до выдачи финального ответа.

## Текущий стек

- Python
- `lmstudio` как API-слой для работы с моделью
- локальный запуск модели через `LM Studio`
- модульная архитектура с использованием ООП
- structured output через JSON schema

Модель по умолчанию:

- `llama-3-8b-gpt-4o-ru1.0`

## Архитектура

Сейчас проект разбит на три основных слоя.

### 1. Агенты

Расположены в `mars_app/agents/`

- `shield.py` - безопасность входа и нормализация пользовательского запроса
- `planner.py` - глобальная декомпозиция задачи
- `manager.py` - управление выполнением этапов
- `worker.py` - исполнение конкретной задачи
- `critic.py` - контроль качества и возврат на доработку
- `editor.py` - финальное оформление результата

### 2. Сервисы

Расположены в `mars_app/services/`

- `memory.py` - память в оперативном виде: контекст, успешные результаты, ошибки
- `web_search.py` - интерфейс-заглушка для будущего подключения веб-поиска

### 3. Ядро системы

Расположено в `mars_app/`

- `llm_client.py` - обёртка над `LM Studio`
- `contracts.py` - схемы structured output для агентов
- `models.py` - общие модели данных
- `prompts.py` - системные промпты для ролей
- `orchestrator.py` - основной управляющий pipeline
- `bootstrap.py` - сборка системы и точка инициализации
- `settings.py` - настройки модели и параметров запуска

Точка входа:

- `Test_LM.py`

## Поток выполнения

Запрос проходит через следующий pipeline:

`User -> Shield -> Planner -> Manager -> Worker -> Critic -> Editor`

Если `Critic` отклоняет результат `Worker`, задача возвращается в `Manager`, который перестраивает постановку с учётом ошибок и запускает новую попытку.

## Текущее состояние проекта

В репозитории уже есть рабочий архитектурный каркас:

- модульная структура из нескольких файлов
- разделение ролей агентов
- orchestrator с повторными попытками
- интерфейс памяти
- заглушка для веб-поиска

Пока ещё не реализовано:

- постоянная память
- реальный backend для веб-поиска
- фильтрация и ранжирование источников
- модели модерации
- система оценки качества и бенчмарков

## Ближайший roadmap

1. Подключить реальный веб-поиск с фильтрацией источников.
2. Добавить постоянную память для фактов проекта и истории ошибок.
3. Усилить защиту через отдельные механизмы модерации и anti-injection.
4. Добавить логирование, трассировку и оценку качества агентов.
5. Подготовить удобный путь для разработки на Windows и развёртывания на Linux.

## Примечание

Проект изначально строится вокруг `LM Studio`, и этот локальный API-слой предполагается сохранить как основную точку работы с моделями.
